Frequently, there is a need to showcase multiple alerts, particularly those at the application level positioned at the top. The clr-extension offers two highly beneficial components for this purpose: AlertsComponent and AlertsService.

Usage

Basic Usage

  1. Import AlertsComponent and include <clx-alerts> above the Clarity header tag within the main container.
  2. Import AlertsService and utilize its addAlert(), deleteAlert(), or clearAlerts() methods for effective management of alerts.

Import the Alert class from clr-extension to instantiate an alert. The first parameter is the alert content in string format, while the second parameter is an optional config object enabling you to specify properties such as alertType and whether it is at the application level.

Advanced Usage

At times, you might find the need to incorporate a button or link within an alert's content. Managing the onTargetClick for button clicks or navigating to an internal route through link clicks involves specifying the onTargetClick function and targetId in the second options parameter. The targetId corresponds to the ID of the clickable element, while the onTargetClick denotes the function to be executed upon clicking the element.
